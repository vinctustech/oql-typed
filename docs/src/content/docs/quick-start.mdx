In five minutes you'll have a typed schema, a working OQL instance, and your first compile-time checked query.

<Steps>

1. **Install the library and a backend.**

   ```bash
   npm install @vinctus/oql-typed @vinctus/oql-pg
   ```

2. **Define your schema in TypeScript.**

   The examples throughout these docs use a small blog domain: `user`, `post`, `category`, `tag`, `comment`. Wrap the whole thing in `defineSchema(...)` so TypeScript preserves the literal entity names.

   ```typescript
   // schema.ts
   import {
     defineSchema, entity,
     uuid, text, integer, boolean, timestamp,
     manyToOne, oneToMany, manyToMany, enumType,
   } from '@vinctus/oql-typed'

   export type UserRole = 'ADMIN' | 'AUTHOR' | 'READER'
   export type PostStatus = 'DRAFT' | 'PUBLISHED' | 'ARCHIVED'

   export const schema = defineSchema({
     user: entity('users', {
       id:          uuid().primaryKey(),
       email:       text(),
       firstName:   text().column('first_name'),
       lastName:    text().column('last_name'),
       role:        enumType<UserRole>('UserRole', ['ADMIN', 'AUTHOR', 'READER']),
       enabled:     boolean(),
       lastLoginAt: timestamp().column('last_login_at').nullable(),
       posts:       oneToMany('post'),
     }),

     post: entity('posts', {
       id:        uuid().primaryKey(),
       title:     text(),
       slug:      text(),
       status:    enumType<PostStatus>('PostStatus', ['DRAFT', 'PUBLISHED', 'ARCHIVED']),
       viewCount: integer().column('view_count'),
       createdAt: timestamp().column('created_at'),
       author:    manyToOne('user', { column: 'author_id' }),
     }),
   })
   ```

   The **entity name** (`'user'`, `'post'`) is the key in `defineSchema(...)`. Relations reference other entities by that same string literal — no thunks, no circular imports.

3. **Instantiate OQL and wrap it.**

   OQL itself runs off a `.dm` data-model string. The recommended workflow is to keep `schema.dm` alongside `schema.ts` and load it at startup. You can bootstrap `schema.ts` from `schema.dm` with the [`oql-typed-codegen` CLI](/oql-typed/guides/codegen/).

   ```typescript
   // db.ts
   import { readFileSync } from 'node:fs'
   import { typedOQL } from '@vinctus/oql-typed'
   import { OQL_PG } from '@vinctus/oql-pg'
   import { schema } from './schema.js'

   const dm = readFileSync(new URL('./schema.dm', import.meta.url), 'utf8')
   const oqlInstance = new OQL_PG(dm, host, port, database, username, password)

   export const db = typedOQL(oqlInstance, schema)
   ```

4. **Write typed queries.**

   ```typescript
   import { db } from './db.js'

   // PK lookup with .findById — auto-terminates with .one()
   const u = await db.user
     .select('id', 'firstName', { posts: ['id', 'title'] })
     .findById(userId)
   // u is typed as:
   //   { id: string, firstName: string, posts: { id: string, title: string }[] } | undefined
   ```

5. **Let the compiler catch mistakes.**

   <Tabs>
     <TabItem label="✗ Typo in a field">
       ```typescript
       db.user.select('id', 'fistName')
       //                    ^^^^^^^^
       // Type error: Argument not assignable to keyof user fields
       ```
     </TabItem>
     <TabItem label="✗ Wrong value type">
       ```typescript
       eq(db.user.enabled, 'yes')
       //                  ^^^^^
       // Type error: string not assignable to boolean
       ```
     </TabItem>
     <TabItem label="✗ Invalid enum">
       ```typescript
       eq(db.user.role, 'SUPERADMIN')
       //               ^^^^^^^^^^^^
       // Type error: not assignable to 'ADMIN' | 'AUTHOR' | 'READER'
       ```
     </TabItem>
     <TabItem label="✗ Unselected field">
       ```typescript
       const u = await db.user.select('id').one()
       u?.firstName
       //  ^^^^^^^^^
       // Type error: 'firstName' not in projection
       ```
     </TabItem>
   </Tabs>

</Steps>
