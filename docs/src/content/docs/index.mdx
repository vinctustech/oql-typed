## Why oql-typed

<CardGrid>
  <Card title="Typed projections" icon="approve-check">
    `.select('id', { account: ['id', 'name'] })` produces a precisely typed result — no manual type parameter, no casts.
  </Card>
  <Card title="Typed filters" icon="approve-check">
    `eq`, `ilike`, `inList`, `exists`, etc. validate field types at compile time. `eq(user.enabled, 'yes')` is a type error.
  </Card>
  <Card title="Schema as source of truth" icon="document">
    Declare entities once. Derive TypeScript types, generate the OQL `.dm` data model, and share the schema across queries and mutations.
  </Card>
  <Card title="Backend-agnostic" icon="setting">
    Works with `@vinctus/oql-pg` (PostgreSQL) and `@vinctus/oql-petradb` (in-memory). Peer dependency — no hard-coded backend.
  </Card>
</CardGrid>
